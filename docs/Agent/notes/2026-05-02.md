## Security Analysis Web View — Design Document

Wrote `docs/Feature Development/Security Analysis Web View.md` — comprehensive design document for the new Security Analysis web view.

### Document contents
- Layout mockups (empty state, company loaded with summary tiles, historical table, charts tab, search dropdown)
- All four required features specified:
  1. Search bar with ticker/edinet code/company name search (reuses existing `search_securities()`)
  2. Latest price + computed key metric tiles (PE, PB, MCap, Div Yield, ROE, P/S, EV/EBIT, D/E) using existing `get_security_overview()`
  3. Historical data table with dynamic column filter (multi-select grouped by source table, period count selector, CSV export)
  4. Flexible charts (Chart.js, line/bar/area, multi-series, multiple chart panels)
- API specification: 11 endpoints (9 reusing existing backend functions, 2 new)
  - `GET /api/security/available-columns` — dynamic column discovery per company
  - `GET /api/security/chart-data` — clean series data for chart rendering
- Architecture decisions: schema-agnostic design, Chart.js for charts, sessionStorage for state persistence
- Five-phase implementation plan: Backend API → Core Shell → Historical Table → Charts → Polish
- Appendix with column ID convention (`Table.Column` qualified format)

### Key design decisions
- The historical table is a unified cross-tab (not per-source tables like the Tk UI) — flatter, less scrolling, user-customizable columns
- Columns are discovered dynamically via `resolve_schema()` — no hardcoded lists
- Value types inferred from column name heuristics (same as screening FORMAT_RULES)
- The existing `get_security_statements()` can be used for table data with client-side pivoting (no new backend endpoint needed for the table)
- The `available-columns` and `chart-data` endpoints are the only new backend work required
- Peer comparison tab deferred to follow-up after core view

## Security Analysis — Round 3 Fixes

### Fix 1: Summary metric tiles
- `computeMetrics` now reads values directly from `valuation_latest` and `quality_latest` in the overview payload (backend already computes these via `_compute_ratio_payload`)
- Removed broken fallback code that tried to read non-existent fields from `fundamentals_latest` (which only has 6 specific fields: Revenue, OperatingIncome, NetIncome, TotalAssets, ShareholdersEquity, SharesOutstanding)
- Added lightweight fallback for MarketCap (`price × SharesOutstanding`) and ROE (`NetIncome / ShareholdersEquity`) using the actual field names from fundamentals_latest

### Fix 2: Metric visibility toggles + remove buttons
- Each metric row now has a checkbox in the first column — uncheck to hide, re-check to show
- Hidden metrics persist in `state.hiddenMetrics` (per-table Set, stored in sessionStorage)
- Each visible metric row has an × button in a new actions column — clicking it hides that metric
- Chart view respects hidden metrics (only visible metrics are plotted)

### Fix 3: All database tables discovered
- `/api/security/history` now iterates over ALL keys in the `get_security_statements` response instead of a hardcoded list of `known_source_keys`
- Unknown tables get auto-generated display names via title-casing
- Playwright test verifies table tabs exist and switching works
