# 2026-04-30

## Screening View — Complete Redesign

### Problem
Initial implementation had poor UX:
- Full DOM rebuild on every table/mode change → flicker & lost focus
- Hundreds of raw checkboxes for columns with no search
- Cramped forms showing everything at once
- No progressive disclosure or visual hierarchy

### Redesign: Compact Cards + Modal + Searchable Selector Layout

**New layout (top→bottom):**
1. **Top bar**: Database picker, date/period controls, Load/Save/Run actions
2. **Criteria bar**: Compact summary cards showing `Table.Column OP Value` with ✎/✕ controls. Click to expand inline editor. + Add opens step-by-step modal.
3. **Selectors row**: Three-column grid — searchable column pool, formula checkboxes, ranking section
4. **Results**: Full-height sortable table with formatted values

**Key UX improvements:**

| Before | After |
|--------|-------|
| Criteria as raw form rows, full list re-renders on every interaction | Compact summary cards, inline editing targets only one criterion at a time |
| All columns shown as raw checkbox list | Searchable pool grouped by table, selected columns shown as removable tags |
| No add-criterion flow | Step-by-step modal: Table → Column → Operator → Mode → Value |
| No query feedback | Query info line shows `period=2024 · date≤2020-01-01 · 3 filters` |
| Sidebar + results split | Top configuration + bottom results (results get 70%+ of space) |

**Technical changes:**
- Criterion editor uses targeted re-render — only the edited card expands, rest stay as summaries
- Column selector has search-as-you-type filter, grouped by table, selected as tags
- Modal with 4-step wizard for adding criteria: Table → Column → Operator/Mode → Values
- All form elements built with `el()` helper, event listeners attached inline
- Static HTML is minimal (root container + modal); everything else built by JS
