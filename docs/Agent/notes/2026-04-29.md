## Architecture note
The frontend is multi-page: each screen lives in its own folder with an HTML file, a page-specific JS entry point, and an `index.js` logic module. Tabs use inline `onclick` for navigation. Server routes in `server.py` serve each HTML file at its own path (`/`, `/orchestrator`, `/screening`, `/security`).
