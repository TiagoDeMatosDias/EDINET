# 2026-05-01

## Screening View — Custom Columns + Collapsible Details

### Feature 1: Custom (computed) columns

The backend (`build_screening_query`, `run_screening`, API routes) already fully supported `computed_columns`. The frontend simply wasn't using them.

**Frontend** (`index.js`):
- Added `computedColumns` and `prebuiltFormulas` to `ST` state
- New "Computed Columns" section in the config area with "+ Add Computed Column" button
- Builder popup with two tabs:
  - **Pre-built**: Loads formulas from `/api/screening/formulas` (P/E, P/B, P/S, Dividend Yield, Earnings Yield). Click to add.
  - **Custom**: Name, formula type (Price Ratio or Custom SQL), numerator/denominator table+column selectors. For Custom SQL, a textarea for the expression.
- Added `renderComputedColumns()` — displays computed column bars with name and hint, removable with ✕
- `run()`, `save()`, `load()`, `exportCSV()` all pass computed columns through to the API

**Backend** (`screening.py`):
- `save_screening_criteria()` now accepts `computed_columns` parameter and persists them in the JSON
- `export_screening_to_backtest_csv()` now accepts `computed_columns` parameter

**API** (`api/screening.py`):
- Save route now passes `computed_columns` to `save_screening_criteria()`
- Export route (both CSV and backtest branches) now passes `computed_columns` through
- `screening_date` is now persisted in save data for future use

**CSS** (`styles.css`):
- `.scr-details` / `.scr-details-toggle` / `.scr-details-body` — collapsible details wrapper
- `.scr-col-comp` / `.scr-tok-comp` / `.scr-hint` — computed column bar styles (purple accent)
- `.scr-pop-comp` / `.scr-comp-tabs` / `.scr-comp-tab` / `.scr-comp-body` etc. — builder popup styles

### Feature 2: Collapsible screening details

The screening config area (date, criteria, computed columns, columns) is now wrapped in a `<details>` element with a "Screening Details ▾" summary — same UX as the SQL section at the bottom. The Run/Save/Load buttons and Export toolbar remain always visible.

- `<details id="scr-details" open>` wrapped around the date + criteria + computed columns + columns sections
- Toggle event handler updates summary text between "▾" (open) and "▸" (closed)
- Custom `::-webkit-details-marker` suppression for clean look
