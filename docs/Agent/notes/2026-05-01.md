## Round 2 — Drag-and-drop column reorder + session cache

### Feature 3: Drag-and-drop column reordering (no re-run)

Column bars (both regular columns and computed columns) now have a drag grip (⠿) and support HTML5 drag-and-drop to reorder. Reordering does NOT trigger a new API call — it only rearranges the existing results table via `reorderResults()`.

**Implementation:**
- `_makeDraggable(bar, kind, idx, onReorder)` — shared drag-and-drop helper
- `renderColumns()` — unified rendering for both regular and computed columns
- `reorderResults()` — reorders `ST.results.columns` and `ST.results.rows` to match `ST.columns` order

**CSS additions:**
- `.scr-col-grip` — drag handle, cursor:grab
- `.scr-col.is-dragging` / `.scr-col.is-over` — drag states
- `.scr-table th.is-dragging` / `.scr-table th.is-over` — header drag states

### Feature 4: Session cache (survives page navigation)

Uses `sessionStorage` to persist state across page navigations. On return to the screening view, previous results and configuration are restored instantly. Metrics are refreshed in the background.

**Implementation:**
- `cacheState()` / `restoreCachedState()` 
- `init()` tries cache first, renders cached results immediately
- `run()` caches after successful response
- `beforeunload` listener as backup

## Round 3 — Unified columns section + header drag

### Change 1: Unified Columns section

Merged the "Computed Columns" and "Columns" sections into a single "Columns" section. All columns (regular + computed) are in one drag-reorderable list. Two add buttons: "+ Add Column" (column picker) and "+ Add Computed" (computed column builder).

**Refactored state**: `ST.columns` is now a unified array where each entry has a `kind` field (`'col'` or `'comp'`). Removed `ST.computedColumns`.

**Helper functions added**:
- `colRefs()` — returns array of `"Table.Column"` strings for kind='col' entries
- `computedColSpecs()` — returns array of API specs for kind='comp' entries

**All references updated**: `run()`, `save()`, `load()`, `exportCSV()`, `showColPicker()`, `showComputedColBuilder()`, `renderColumns()`, `reorderResults()`, `cacheState()`, `restoreCachedState()`, `setDefaultColumns()`

### Change 2: Drag column headers in results table to reorder

Results table headers (`<th>`) are now draggable. Dropping a header reorders the results columns + rows and syncs the config section's column order.

**Drop handler logic**:
- Moves the result column + all row data from source to target position
- Finds matching config entries for both the moved column and the target column
- If both have config entries: places the moved config item adjacent to the target
- If the target has no config entry (e.g. LatestPrice, PriceDate): places near neighbors
